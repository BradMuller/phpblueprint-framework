simple, flexible, expandable
  • Naming Conventions:
    1. All files except view files must start with an uppercase followed by all lowercase.
    2. All controller classes must have the suffix '_Controller' and follow rule 1.
    -  Example, 'MyController' in the file 'Mycontroller.php' is 'class Mycontroller_Controller extends Controller'.
    3. The same is done for models as for controllers in rule 2.
    -  Example, 'MyModel' in the file 'Mymodel.php' is 'class Mymodel extends Model'.
    4. Library class names must follow rule 1.
  • All configuration file's $config variable's will be loaded into the parent model and controller classes as objects.
  • To access the configuration variable '$config['one']['two']' inside a model or controller use: 
    '$this->config->one->two'
  • Controller classes extend 'Controller' and Model classes extend 'Model'. Don't forget to call the parent constructor to get the 'load' and 'config' objects.
  • To load a library object use: 
    '$this->load->library('my_library')'
  • To load a model use 
    '$this->load->model('my_model')'
  • To load a controller use: 
    '$this->load->controller('my_controller')'
  • To load a view use: 
    '$this->load->view('my_view')'
  • An associative array of data can be passed to the view and its key's will be extracted into variables to be used by the view by using: 
    '$this->load->view('my_view', $my_data_array)'
